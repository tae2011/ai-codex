# Spring Boot 与 RAG 示例知识库

## Spring Boot 是什么

Spring Boot 是基于 Spring 生态的快速开发框架，目标是减少繁琐配置，让开发者更快搭建可运行的服务。它常见的能力包括自动配置、内嵌 Web 容器、统一的配置管理和丰富的 starter 依赖。

## 自动配置

Spring Boot 的自动配置核心思想是按需装配。框架会根据当前 classpath、配置文件和 Bean 条件自动注册组件。开发者可以通过 `@SpringBootApplication` 启动应用，也可以通过 `@ConditionalOnClass`、`@ConditionalOnMissingBean` 等条件注解理解自动配置的生效逻辑。

## RAG 的基本思路

RAG 是 Retrieval-Augmented Generation，也就是“检索增强生成”。它通常包含三个步骤：

1. 先把知识文档切片。
2. 再用嵌入模型把切片转成向量。
3. 用户提问时，先检索最相关的文档片段，再把结果与用户问题一起交给大模型生成答案。

## 为什么 RAG 有用

RAG 的优势在于可以让大模型回答更贴近私有知识库，而不是只依赖通用训练知识。它还可以降低幻觉风险，并让回答更容易追溯到具体资料来源。

## Markdown 导入建议

如果你准备把 Markdown 作为知识库导入，建议注意以下几点：

- 标题层级尽量清晰。
- 每个章节聚焦一个主题。
- 长段落可以适当拆分。
- 对重要术语给出完整表述。

## 本项目里的使用方式

在这个项目里，Markdown 文档会先导入知识库，再由嵌入模型生成向量。后续聊天时，系统会先检索本地向量库，找到最相关的片段，并把这些片段拼进系统上下文，再调用大模型生成回答。
